## 一、架构概述：V2.0 扁平化设计原则

### 1.1 去套娃与极致压缩

V2.0 架构已完成以下关键重构：

- **消除双重嵌套**：彻底清理了 `ai外挂工程/ai外挂工程` 的套娃结构，所有代码现在位于单一的 `ai外挂工程/` 目录下
- **去除 Pyppeteer 依赖**：完全移除了过时的 Pyppeteer，统一使用 Playwright 作为浏览器自动化引擎
- **模块扁平化**：三个核心模块（记忆内核、安全守卫、爬虫引擎）均采用扁平化目录结构

### 1.2 单一可信配置源

**核心原则**：子目录私有 `.env` 已物理删除，所有脚本统一通过 `find_dotenv(use_required=True)` 向上追溯加载根目录下的唯一 `.env`。

```
项目根目录/
├── .env                    ← 【唯一配置源】所有脚本从此文件读取配置
├── ai外挂工程/
│   ├── setup.ps1           ← 一键部署脚本
│   ├── 01_memory_core/     ← 记忆内核模块
│   ├── 02_git_defender/    ← Git 安全守卫模块
│   └── .env.example        ← 配置模板（仅供参考，实际配置在根目录）
└── ...
```

### 1.3 元规则自适应注入

`setup.ps1` 实现了一键自动向多 IDE 注入脑干级记忆规则：

| IDE/工具 | 注入目标 | 注入内容 |
|----------|----------|----------|
| **Trae** | `.trae/rules/memory-agent.md` | 记忆分层架构规范 |
| **VS Code / Copilot** | `.vscode/copilot-instructions.md` | 项目开发规则 |
| **Cursor** | `.cursorrules` | 智能体行为规范 |
| **Claude Code** | `CLAUDE.md` | 项目指令与记忆管理 |

### 1.4 双向安全隔离

#### Git 隔离
- 大项目 `.gitignore` 屏蔽 `ai外挂工程/`，防止套件代码污染主项目
- 外挂内部无 `.git` 子目录，避免嵌套仓库冲突

#### TSC 隔离（适用于 TypeScript 项目）
- 大项目 `tsconfig.json` 的 `exclude` 块中排除以下路径：
  - `ai外挂工程`
  - `.venv`
  - `.pip-cache`

### 1.5 免代理降级抓取

基于 Python 标准库 `urllib` 实现的在线降级抓取策略：

1. **直接连接**：优先尝试直接访问目标 URL
2. **云端网关降级**：若直接失败，自动依次尝试以下免费 CORS 代理：
   - `api.allorigins.win`
   - `api.codetabs.com`
   - `corsproxy.io`
3. **零配置**：无需本地代理软件，无需配置 `HTTP_PROXY` 环境变量

## 三、核心模块说明

### 3.1 记忆内核 (`01_memory_core/`)

**功能**：为 AI 智能体提供长期记忆能力，支持断点续传和自我接力开发。

**核心文件**：
- `utils.py`：工具库，包含 `load_env_config()`、`fetch_html()`、`call_llm()` 等
- `archive_chronicle.py`：记忆归档脚本，自动压缩和归档历史日志
- `crawl_test.py`：爬虫测试脚本，验证云端网关降级抓取

**配置规范**：
- `SPEC.md`：项目宪法，存放静态技术栈和目录结构
- `CHRONICLE.md`：滚动日志，记录最新任务活动断点（窗口上限 5 条）
- `logs/archive/`：历史归档目录，按月存储压缩后的记忆

**自动落盘机制**：
当 `CHRONICLE.md` 超过 5 条记录时，自动触发：
1. 语义压缩最老的 3 条记录
2. 将压缩结果追加到 `SPEC.md` 的"历史里程碑"章节
3. 原始日志移动到 `logs/archive/mem_YYYY_MM.md`
4. 保持 `CHRONICLE.md` 窗口仅保留最新 2~3 条

### 3.2 Git 安全守卫 (`02_git_defender/`)

**功能**：防止敏感信息（API Key、密码）通过 Git 提交泄露。

**激活方式**：
```powershell
git config core.hooksPath ai外挂工程/02_git_defender/githooks
```

**工作原理**：
- `pre-commit`：Git 钩子脚本，在每次提交前自动触发
- `pre-commit-audit.py`：审计脚本，扫描暂存区中的敏感信息

### 3.3 爬虫引擎（集成在 `01_memory_core/`）

**功能**：无需本地代理即可抓取海外网站，自动提取动效信息并生成报告。

**技术栈**：
- **Playwright**：替代 Pyppeteer 的现代浏览器自动化引擎
- **urllib**：Python 标准库，用于免代理 HTTP 请求
- **BeautifulSoup4**：HTML 解析

**输出**：
- 爬取报告保存在 `reports/` 目录
- 自动检测并记录页面中的动画库（GSAP、Lottie、anime.js 等）

## 四、迁移指南：将套件部署到新项目

### 4.1 最小迁移方案

只需复制以下目录到新项目根目录：

```
ai外挂工程/
├── 01_memory_core/     ← 记忆内核
├── 02_git_defender/    ← Git 安全守卫
├── setup.ps1           ← 部署脚本
├── .env.example        ← 配置模板
└── requirements.txt    ← Python 依赖
```

### 4.2 部署后配置

1. **运行部署脚本**：
   ```powershell
   powershell -ExecutionPolicy Bypass -File .\ai外挂工程\setup.ps1
   ```

2. **配置 API Key**：
   打开根目录 `.env`，填入您的 LLM API Key

3. **绑定 Git 钩子**：
   ```powershell
   git config core.hooksPath ai外挂工程/02_git_defender/githooks
   ```

4. **初始化记忆**：
   在新项目根目录创建空白 `MEMORY.md`，然后运行 AI 记忆初始化提示词

### 4.3 TypeScript 项目集成

在 `tsconfig.json` 中添加排除规则：

```json
{
  "exclude": [
    "ai外挂工程",
    ".venv",
    ".pip-cache"
  ]
}
```

在 `.gitignore` 中添加：

```
ai外挂工程/
```

## 五、常见问题

### Q1: 为什么子目录下没有 `.env` 文件？
**A**: V2.0 架构采用单一配置源原则，所有 `.env` 已统一到项目根目录。子目录脚本通过 `find_dotenv()` 自动向上追溯加载。

### Q2: 为什么不再使用 Pyppeteer？
**A**: Pyppeteer 已停止维护，Playwright 提供了更好的稳定性和跨浏览器支持。

### Q3: 如何验证抓取功能是否正常？
**A**: 运行 `.\.venv\Scripts\python.exe ai外挂工程/01_memory_core/crawl_test.py`，观察是否能成功抓取 Hacker News。

### Q4: `setup.ps1` 可以重复运行吗？
**A**: 可以。脚本会自动检测已存在的配置并跳过，不会覆盖您的 `.env` API Key。
