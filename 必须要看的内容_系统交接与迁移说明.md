亲爱的开发者（或未来的 AI 智能体）：
当你看到这个文件时，意味着你已经进入了我们的"智能体沙箱武器库"。为了保证系统的可持续运行，以及你在未来将它们移植到新项目时能够无缝、安全地进行，请务必严格阅读并遵守本指南。

## 一、 核心结论：下次迁移，你只需要动这三个"手提箱"！

本套件采用了极致解耦的模块化设计。未来你在新项目中需要任何自动化功能时，**只需要复制对应文件夹内部的所有内容**到你的新项目根目录下即可：

1.  **`01_memory_core/`** (🎁 【手提箱一：长期记忆内核】)
    *   *何时迁移：* 任何需要 AI 具备"断点续传"、"自我接力开发"能力的 React/Vue/Python 新项目。
2.  **`02_git_defender/`** (🎁 【手提箱二：Git安全守卫】)
    *   *何时迁移：* 任何包含了敏感 API Keys、密码、且需要推送到公共 GitHub 仓库的项目。
3.  **`03_browser_crawler/`** (🎁 【手提箱三：自愈式多模态爬虫】)
    *   *何时迁移：* 需要无人值守自动爬取网页、提取动效灵感、生成总结报告且自动监控 Token 成本的项目。

## 二、 根目录下那些"不用动"的东西，为什么绝对不能迁移？

根目录外侧平铺的几个文件/文件夹（如 `.git`、`.gitignore`、`.trae`）属于**沙箱环境的专属底盘**，请把它们留在里面，**绝对不要直接复制覆盖到新项目**：

*   **`.git/`（版本控制库）：**
    *   *不迁移原因：* 每个新项目都有自己独立的本地代码提交历史（新项目运行 `git init` 自动生成）。如果你把这个沙箱的 `.git` 拷过去，会直接损坏并覆盖新项目的提交历史。
*   **`.gitignore`（忽略配置文件）：**
    *   *不迁移原因：* 新项目有它自己前端（React/Vue/Node）的一套忽略规则。如果直接覆盖，会导致新项目的忽略规则丢失。
    *   *正确用法：* 用**记事本**打开它，复制最底下那 3 行智能体专属忽略规则（如 `config/settings.yaml`），追加粘贴到新项目 `.gitignore` 的最底部。
*   **`.trae/`（外侧运行缓存）：**
    *   *不迁移原因：* 这是你在当前沙箱窗口中运行 Trae 软件时，由 Trae 自动生成的本地技能与设置缓存。迁移它没有任何实际工程意义。

## 三、 迁移这三个模块时，分别有哪些地方需要修改/配置？

每一个"手提箱"都有独立的说明书（`迁移与配置说明.txt`），当把它们的内容复制到新项目根目录后，你需要且仅需要执行以下关键修改：

### 1. 迁移 `01_memory_core` 后：
*   **需要新建：** 在新项目根目录下新建一个空白的、0 KB 的 **`MEMORY.md`** 文件（沙箱里这个不需要拷过去，让新项目保持空白）。
*   **需要命令：** 在新项目的 Trae SOLO 窗口中输入老项目记忆初始化的提示词，让 AI 自动审计新项目的代码，并重新格式化写入它的第一篇记忆。

### 2. 迁移 `02_git_defender` 后：
*   **需要命令：** 以管理员身份在新项目的控制台/终端运行以下命令，激活本地 Git 钩子绑定：
    ```bash
    git config core.hooksPath githooks
    ```

### 3. 迁移 `03_browser_crawler` 后：
*   **需要填入：** 打开新拷贝过去的 `.env` 文件，在 `LLM_API_KEY=` 后面填入您实际的小米 mimo-v2.5 的 API Key。
*   **需要安装：** 在终端中进入 `services/browser_distillation`，运行 `uv sync` 构建好独立的 Python 依赖虚拟环境。
*   **需要挂载：** 以管理员身份运行 `register_cron.ps1`，自动向 Windows 注册每日定时爬取数据的计划任务。
