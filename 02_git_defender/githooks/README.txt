========================================
githooks Git 钩子文件夹说明
========================================

【文件夹作用】
本文件夹是本地 Git 提交拦截器（安全防护网），用于在代码提交前自动进行安全检查。

----------------------------------------
文件清单
----------------------------------------

1. pre-commit           - Shell 引导脚本（负责找到 Python 解释器并调用审计脚本）
2. pre-commit-audit.py  - Python 审计脚本（执行实际的安全检查逻辑）

----------------------------------------
功能说明
----------------------------------------

每次执行 git commit 时，钩子会自动运行以下检查：

【检查 1：敏感信息检测】
- 扫描暂存区中的 config/settings.yaml 文件
- 检测 api_key 字段是否包含真实 API 密钥
- 如果检测到真实密钥，立即中断提交并发出红色警告
- 防止 API 密钥被意外提交到代码仓库

【检查 2：Python 语法检查】
- 对暂存区中所有变动的 .py 文件执行语法检查
- 使用 Python 的 py_compile 模块验证语法正确性
- 如果发现语法错误，阻止提交并提示修复

----------------------------------------
激活命令
----------------------------------------

在项目根目录下运行以下命令即可一键启用：

  git config core.hooksPath githooks

运行后，Git 会优先读取本文件夹下的钩子脚本。

----------------------------------------
迁移说明
----------------------------------------

将本钩子迁移到其他项目的步骤：

1. 复制整个 githooks 文件夹到新项目的根目录
2. 在新项目根目录下运行激活命令：
   git config core.hooksPath githooks
3. 确保新项目中存在 config/settings.yaml 文件（否则敏感信息检测会被跳过）
4. 确保新项目的 Python 虚拟环境路径正确（脚本会自动检测 .venv/Scripts/python.exe）

----------------------------------------
技术细节
----------------------------------------

- pre-commit 脚本会按以下顺序查找 Python 解释器：
  1. 当前项目 .venv/Scripts/python.exe（推荐）
  2. 系统 PATH 中的 python 命令
  3. 系统 PATH 中的 python3 命令
  4. 如果都找不到，报错退出

- 使用本地虚拟环境可以避免对系统 Python 的依赖

----------------------------------------
测试钩子
----------------------------------------

测试敏感信息拦截：
  git add config/settings.yaml
  git commit -m "test"
  # 应该看到红色警告并被阻止提交

测试正常提交：
  git add githooks/
  git commit -m "test hooks"
  # 应该看到绿色的 PASSED 提示
