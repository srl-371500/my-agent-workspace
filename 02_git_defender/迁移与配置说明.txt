========================================
02_Git提交防泄密拦截器 模块说明
========================================

【模块作用】
本模块是 Git 提交前的安全拦截器，用于防止敏感信息（如 API 密钥）被意外提交到代码仓库。

核心功能：
- 检测 config/settings.yaml 中的真实 API 密钥
- 对暂存区的 Python 文件进行语法检查
- 如果检测到安全风险，自动阻止提交

----------------------------------------
Git 安全拦截模块迁移指南
----------------------------------------

请按以下步骤将本模块迁移到新项目：

1. 将本文件夹内的 githooks 和 config 文件夹复制到新项目根目录下

2. 在新项目的 .gitignore 中加入以下内容：
   config/settings.yaml

3. 在终端运行以下命令一键启用提交前密钥审计防护：
   git config core.hooksPath githooks

4. 测试钩子是否生效：
   - 将一个包含真实 API 密钥的 settings.yaml 放入暂存区
   - 执行 git commit -m "test"
   - 应该看到红色警告并被阻止提交

----------------------------------------
文件说明
----------------------------------------

githooks/pre-commit
  - Shell 引导脚本，负责找到 Python 解释器并调用审计脚本

githooks/pre-commit-audit.py
  - Python 审计脚本，执行实际的安全检查逻辑

config/settings.yaml
  - API 密钥配置模版，api_key 字段留空
  - 仅供 pre-commit 脚本读取校验使用

----------------------------------------
激活与停用
----------------------------------------

激活钩子：
  git config core.hooksPath githooks

停用钩子（恢复默认）：
  git config --unset core.hooksPath

查看当前钩子路径：
  git config core.hooksPath
