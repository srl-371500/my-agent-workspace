# 分层记忆内核行为规范

## 1. 记忆分层架构
- **SPEC.md**：项目宪法，只存放静态技术栈、目录结构和核心契约（1 页纸）
- **CHRONICLE.md**：滚动日志，只记录最新任务活动断点（滚动窗口）
- **logs/archive/**：历史归档，按月压缩存储（`mem_YYYY_MM.md`）

## 2. 自动落盘约束规则
当 `CHRONICLE.md` 的活动任务记录超过 **5 条**时，你必须在终端中启动自动落盘逻辑：

### 2.1 语义压缩
- 提取最老的 **3 条**任务记录
- 使用 LLM 进行语义高度压缩，提炼为一条"历史里程碑"
- 追加到 `SPEC.md` 的"历史里程碑"章节

### 2.2 原始归档
- 将被压缩的 3 条原始详细日志，剪切并移动到 `logs/archive/mem_YYYY_MM.md`
- 按当前年月创建或追加归档文件

### 2.3 窗口清理
- 清空并保持 `CHRONICLE.md` 中只有最新的 **2~3 条**活动断点
- 更新任务统计指标

## 3. 日常维护规范
- 每次任务完成后，更新 `CHRONICLE.md` 的活动任务窗口
- 严禁在 `SPEC.md` 中写入动态日志
- 严禁在 `CHRONICLE.md` 中写入静态技术契约

## 4. 用户提示
在本轮任务收工、代码提交或完成重要功能后，主动向人类提示：
"本次任务已完成，我已为你更新了 CHRONICLE.md 中的活动断点记录，随时可以查看。"
